# 浴室歌手 (Bathroom Singer) — UI 设计方案

> UI Designer 出品 · 面向「洗澡清唱、记不住歌词」场景的自动滚动歌词网页
> 目标设备：iPad（竖屏优先，兼顾横屏）· 部署：Vercel（纯静态）

---

## 1. 产品定位与核心约束

| 维度 | 需求 | 设计含义 |
|------|------|----------|
| 场景 | 洗澡时清唱，记不住歌词 | 歌词即界面，自动滚动 |
| 设备 | iPad 主力 | 竖屏优先，触控但**不触碰** |
| 视力 | 不戴眼镜 | 超大字号 + 超高对比 |
| 交互 | 无法触控屏幕 | 全程声控（仅首次点一下"开始"） |
| 内容 | 只清唱，无伴奏 | 只展示歌词，无播放器 |
| 数据 | 歌词内置 | 不依赖网络，洗澡时可用 |
| 部署 | Vercel | 纯静态站点，零后端 |

---

## 2. 设计原则

- **极简单屏**：没有多余按钮，歌词本身就是界面。
- **超大字 + 超高对比**：无眼镜也能一眼看清。
- **零触摸交互**：除首次"开始"点击外，全程声控。
- **卡拉 OK 式高亮**：当前行突出、邻行淡化、自动居中滚动。
- **声控可视反馈**：聆听状态 / 识别结果 / 出错提示一目了然。

---

## 3. 设计系统 (Design Tokens)

### 颜色（暗色为主，抗浴室反光眩光）
```css
:root {
  --bg:          #0d1117;  /* 近黑底，省电、减眩光、歌词更跳 */
  --text:        #f0f6fc;  /* 主文本，对 --bg 对比度 ~16:1，超 AAA */
  --text-dim:    #5b636e;  /* 邻行淡化 */
  --text-sub:    #8b949e;  /* 歌名/提示 */
  --voice:       #22d3ee;  /* 语音聆听中（青，脉冲） */
  --accent:      #ffd166;  /* 当前行高亮（暖黄，暗底醒目不刺眼） */
  --ok:          #34d399;  /* 识别成功 */
  --err:         #f87171;  /* 未识别/错误 */
}
```
> 为什么暗色：浴室灯亮 + 屏幕易反光，暗底大字对比强、更护眼，且 iPad 更省电。

### 字体（系统圆体，零加载、离线可用）
- 字体栈：`"PingFang SC", "Hiragino Sans GB", system-ui, sans-serif`
- 歌词字号（语音可调档）：**S 40px / M 52px / L 64px / XL 80px**（默认 M）
- 歌名 20px、副信息 13px、底部提示 14px
- 行高 1.45；当前行字重 700，邻行 400

### 间距（8px 节奏）
- 歌词行间距 ≥ 行高，避免大字拥挤；屏幕四周留白 ≥ 22px

### 动效（尊重 prefers-reduced-motion）
- 当前行切换：`translateY + opacity`，300ms ease（GPU 合成，性能好）
- 聆听脉冲：mic 圆环呼吸 1.4s
- 屏幕常亮：Wake Lock API（Safari 16.4+）

## 4. 界面布局（iPad 竖屏）

```
┌─────────────────────────────┐
│ 正在演唱             ● 聆听中 │  ← 顶部条：歌名/歌手 + 语音状态
│ 月亮代表我的心  邓丽君        │
│                             │
│      你问我爱你有多深        │  ← 上一行（淡化）
│      我爱你有几分            │  ← 当前行（暖黄大字，居中）
│      我的情也真              │  ← 下一行（淡化）
│      我的爱也真              │
│                             │
│  说：下一首 · 暂停 · 下一行   │  ← 底部可收起指令提示
└─────────────────────────────┘
        ▔▔▔▔▔ 极细进度条
```

- **顶部条（极简）**：左歌名+歌手；右语音状态指示（mic 图标，聆听时脉冲青色）。
- **中部（核心）**：当前歌词行大字居中，邻行上下错位并淡化，自动滚动保持当前行垂直居中。
- **底部**：可收起语音指令提示条。
- **进度**：极细进度条（底部边），显示当前歌曲进度。

## 5. 声控设计

### 5.1 首次手势（iOS 必需）
全屏大按钮 **「🚿 点一下，开始唱歌」** —— 满足 iOS 要求"用户手势才能启动语音识别"；点击即请求麦克风权限并进入聆听。这是洗澡前唯一一次触碰。

### 5.2 指令集（中文 `zh-CN`）
| 指令 | 作用 |
|------|------|
| 下一首 / 上一首 | 切歌 |
| 暂停 / 继续 | 暂停或恢复自动滚动 |
| 下一行 / 上一行 | 手动微调当前行 |
| 回到开头 | 从头唱 |
| 字体大一点 / 字体小一点 | 切换字号档位（S→M→L→XL） |
| （可选）这首歌 | 用语音朗读歌名（SpeechSynthesis） |

### 5.3 语音状态机
```
空闲 → 聆听中(脉冲) → 识别到(1.5s 回显"听见了：下一首") → 执行 → 回到聆听
                          │
                          └─ 未识别/错误：轻提示"没听清，再说一次？"，自动继续聆听
```
- 用 **SpeechSynthesis** 朗读确认，闭眼/无眼镜也能感知操作结果（强体验加成）。
- 识别间断自动 `onend` 重启，保证"一直在听"。

## 6. 歌词数据与内置方案

- **格式**：LRC 时间戳（`[mm:ss.xx] 歌词`），解析为带时间的行 → 精准自动滚动。
- **兜底**：无时间戳则按每行固定时长（默认 4s）滚动，可用"下一行"微调。
- **内置**：先放 3–5 首示例（经典老歌），写入 `public/lyrics/*.lrc` 或内联 JSON。
- **扩展**：提供"加歌"说明（把 LRC 文本粘进去即可），全部内置不依赖网络。

---

## 7. 技术架构（部署 Vercel）

- **形态**：纯静态站点（无后端）→ Vercel 零配置部署。
- **推荐**：单一 `index.html + style.css + app.js + lyrics.js`（零构建，最稳）；或 Vite + 原生 TS 便于维护。
- **语音**：Web Speech API（`webkitSpeechRecognition`，`lang=zh-CN`，`continuous` + 自动重启）。
- **常亮**：Wake Lock API。
- **部署**：`git push` → Vercel，或 `vercel` CLI；HTTPS 由 Vercel 提供（语音识别所需）。

---

## 8. 关键风险与对策

| 风险 | 对策 |
|------|------|
| iOS Safari 语音会间断停止 | `onend` 自动重启；首次手势启动（全屏"开始"按钮） |
| 中文识别可用 | `lang=zh-CN`，iOS Safari 支持 |
| 浴室雾气/反光看不清 | 暗底 + 超大字 + 超高对比；可选"屏幕变亮"指令 |
| 语音识别需联网（iOS 将音频发苹果服务器） | 歌词全内置；提示洗澡环境需有 WiFi。完全离线需本地模型，超出本期 |
